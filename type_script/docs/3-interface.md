TypeScriptにおける**インターフェース（Interface）**とは、オブジェクトの**「設計図」**や**「規約（ルール）」**のようなものです。

「このオブジェクトは、必ずこの名前のプロパティを持ち、それはこの型でなければならない」という**形（構造）を定義する**ために使われます。

---

### 1. インターフェースの基本形

例えば、「ユーザー」というデータを扱うとき、バラバラな書き方をされるとプログラムが壊れてしまいます。そこでインターフェースを定義します。

```typescript
// 設計図（インターフェース）の定義
interface User {
  id: number;
  name: string;
  age: number;
}

// 設計図に基づいたオブジェクトの作成
const user1: User = {
  id: 1,
  name: "田中",
  age: 30
};

// 【エラー例】
const user2: User = {
  id: 2,
  name: "佐藤"
  // age がないので、TypeScriptが「設計図通りじゃないですよ！」と警告を出す
};

```

### 2. インターフェースの主な特徴

#### ① 任意プロパティ（Optional Properties）

「あってもなくても良い」項目を `?` で指定できます。

```typescript
interface User {
  id: number;
  name: string;
  email?: string; // emailはあってもなくてもOK
}

```

#### ② 読み取り専用（Readonly）

作成した後に値を書き換えられたくないプロパティに指定します。

```typescript
interface Config {
  readonly apiKey: string;
}

const myConfig: Config = { apiKey: "12345" };
// myConfig.apiKey = "67890"; // エラー！書き換え不可

```

#### ③ 継承（Extends）

既存のインターフェースを元に、新しい機能を追加した設計図を作れます。これがインターフェースの非常に強力な点です。

```typescript
interface Animal {
  name: string;
}

// Animalを継承してDogを作る
interface Dog extends Animal {
  breed: string; // 犬種を追加
}

const myDog: Dog = {
  name: "ポチ",
  breed: "柴犬"
};

```

---

### 3. なぜインターフェースが必要なのか？（メリット）

1. **ミスを防ぐ**: オブジェクトのプロパティ名を打ち間違えたり、必要なデータを入れ忘れたりした際に、実行前に気づけます。
2. **コードが「説明書」になる**: どんなデータ構造なのか、インターフェースを見るだけで理解できます。
3. **エディタの補完が効く**: VS Codeなどのエディタが、次に打つべきプロパティ名を候補に出してくれるようになります。

---

### 「Type Alias（型エイリアス）」との違いは？

TypeScriptには、似た機能に `type`（型エイリアス）があります。

* **Interface**: 主に**「オブジェクトの形」**を定義するのに適しており、同じ名前で定義を後から「拡張（マージ）」できる特性があります。
* **Type**: オブジェクトだけでなく、`string | number` のような**「単純な型や複雑な組み合わせ」**を定義するのにも使われます。

初心者のうちは、**「オブジェクト（データの塊）の形を決めるなら `interface`」**と覚えておけば間違いありません。
