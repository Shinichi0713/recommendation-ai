
目的関数が微分不可能な凸関数である場合勾配を定義できないということになります。
こんなケースで凸関数を最小化したい場合に取りえる方法が劣勾配法（subgradient method）という手法です。

## 劣勾配法

劣勾配法は、**微分不可能な凸関数を最小化するための最適化アルゴリズム**です。  
勾配降下法を「微分不可能な関数」に拡張したもの、と考えるとイメージしやすいです。


### 1. 劣勾配の定義

凸関数 $f$ の点 $x$ における劣勾配集合 $\partial f(x)$ は、

$$
\partial f(x) = \{ g \mid f(y) \ge f(x) + g^\top (y - x)\ \text{for all } y \}
$$

で定義されます。  
これは「すべての方向について一次近似が下から抑えられるような傾きベクトル $g$ の集まり」です。

- 滑らかな点では $\partial f(x) = \{\nabla f(x)\}$
- 微分不可能な点では複数のベクトルを含む（例：$|x|$ の $x=0$ で $[-1,1]$）

>__一次近似が下から抑えられる__
>「一次近似が下から抑えられる」とは、**凸関数のグラフが、その点での接線（より正確には支持超平面）より常に上にある**、という意味です。
>__1. 数式的な意味__
>点 $x$ における劣勾配 $g$ は、すべての $y$ について
>$$
f(y) \ge f(x) + g^\top (y - x)
>$$
を満たすベクトルとして定義されます。
>- 右辺：$f(x) + g^\top (y - x)$ は、点 $x$ を通り傾き $g$ を持つ**一次関数（直線）** です。
>- 左辺：$f(y)$ は実際の関数値です。
この不等式は、
> 「関数値 $f(y)$ は、その点での一次近似（直線）より**常に大きいか等しい**」
>ということを表しています。  
>つまり、一次近似が**下から抑えられている**（下界になっている）ということです。
>__2. 図形的なイメージ__
1次元の凸関数 $f$ を想像してください。
>- ある点 $x$ で接線（傾き $g$）を引きます。
>- 凸関数のグラフは、その接線の**上側**にあります。
>- つまり、接線はグラフの**下側からの支持線（supporting line）** になっています。
この「接線がグラフの下側にある」ことを、「一次近似が下から抑えられる」と表現しています。
>__3. 微分可能な場合との比較__
もし $f$ が微分可能なら、勾配 $\nabla f(x)$ がただ一つ存在し、
>$$
f(y) \ge f(x) + \nabla f(x)^\top (y - x)
>$$
>が成り立ちます（凸関数の性質）。  
このときの一次近似は「接線」そのものです。
>微分不可能な点では、接線が一意に定まらない代わりに、**複数の支持線（劣勾配）** が存在します。  
それらはすべてグラフの下側にあり、「一次近似が下から抑えられる」という性質を満たします。



### 2. 劣勾配法の更新式

劣勾配法は、各ステップで

1. 現在の点 $x_k$ における劣勾配 $g_k \in \partial f(x_k)$ を一つ選ぶ
2. その方向にステップ $\alpha_k$ だけ進む

という更新を行います。

$$
x_{k+1} = x_k - \alpha_k g_k,\quad g_k \in \partial f(x_k)
$$

- $\alpha_k$：ステップサイズ（例：$\alpha_k = \frac{c}{\sqrt{k}}$）
- $g_k$：劣勾配（勾配の一般化）

### 3. 性質・特徴

__(1) 収束はするが遅い__

適切なステップサイズのもとで、劣勾配法は最適解に収束することが知られています。  
ただし、

- 勾配降下法のような「単調減少」は保証されない
- 収束速度は一般に $O(1/\sqrt{k})$ 程度で、勾配法より遅い

__(2) ステップサイズが重要__

ステップサイズ $\alpha_k$ の選び方で収束性が大きく変わります。

- 大きすぎる：発散したり振動する
- 小さすぎる：収束が遅すぎる
- よく使われるルール：$\alpha_k = \frac{c}{\sqrt{k}}$ など

__(3) 実装は単純だが高性能ではない__

- 実装は単純（勾配の代わりに劣勾配を使うだけ）
- しかし、高精度の解や高速収束を求める場面では、近接勾配法・内点法・LP変換など、より高度な手法が好まれます。

### 4. まとめ

- 劣勾配法は、**微分不可能な凸関数に対する勾配法の拡張**。
- 更新式は $x_{k+1} = x_k - \alpha_k g_k,\ g_k \in \partial f(x_k)$。
- 収束はするが遅く、ステップサイズの選び方が重要。
- 理論的には汎用的だが、実用上はより効率的な手法が使われることが多い。

「勾配が定義できない関数でも、劣勾配を使って少しずつ最適解に近づく方法」と覚えておくとよいです。

## 実験

### 勾配法と劣勾配法両方が成立

**滑らかな凸関数**では、勾配法も劣勾配法も両方とも成立します。

### 1. 滑らかな凸関数では勾配＝劣勾配

凸関数 $f$ が微分可能（滑らか）な点 $x$ では、劣勾配集合は

$$
\partial f(x) = \{\nabla f(x)\}
$$

と、**勾配ただ1つ**になります。

このとき、

- **勾配法**：$x_{k+1} = x_k - \alpha_k \nabla f(x_k)$
- **劣勾配法**：$x_{k+1} = x_k - \alpha_k g_k,\ g_k \in \partial f(x_k)$

ですが、$\partial f(x_k) = \{\nabla f(x_k)\}$ なので、**劣勾配法は勾配法と全く同じ更新**になります。

つまり、**滑らかな凸関数に対しては、勾配法は劣勾配法の特別な場合**とみなせます。

### 2. 具体例

__(1) 二次関数（L2損失）__

$$
f(x) = \frac{1}{2}\|Ax - b\|_2^2
$$

は至るところ微分可能な凸関数です。

- 勾配：$\nabla f(x) = A^\top(Ax - b)$
- 劣勾配：$\partial f(x) = \{\nabla f(x)\}$

なので、勾配法でも劣勾配法でも同じ更新式になります。

__(2) L2正則化付き線形回帰__

$$
f(x) = \frac{1}{2}\|Ax - b\|_2^2 + \frac{\lambda}{2}\|x\|_2^2
$$

も滑らかな凸関数です。

- 勾配：$\nabla f(x) = A^\top(Ax - b) + \lambda x$
- 劣勾配：$\partial f(x) = \{\nabla f(x)\}$

したがって、勾配法・劣勾配法のどちらでも同じアルゴリズムとして実装できます。

### 3. 実験

勾配法と劣勾配法でどちらが収束が早いか実験を行います。

以下では、滑らかな凸関数

$$
f(x) = \frac{1}{2}\|Ax - b\|_2^2
$$

に対して、

- 勾配法（gradient descent）
- 劣勾配法（subgradient method）

の両方で更新を行い、目的関数値の収束を比較・可視化する Python コードを示します。  
滑らかな凸関数では勾配＝劣勾配なので、理論的には両者の更新は同じになり、収束曲線も一致します。

### 4. 準備と問題設定

```python
import numpy as np
import matplotlib.pyplot as plt

plt.rcParams["font.size"] = 14
np.random.seed(123)

# 問題サイズ
n = 20
m = 50

# ランダムな行列 A と真の解 x_true
A = np.random.randn(m, n)
x_true = np.random.randn(n)
b = A @ x_true

# 目的関数 f(x) = 0.5 * ||Ax - b||^2
def f(x):
    return 0.5 * np.linalg.norm(A @ x - b)**2

# 勾配（滑らかなので劣勾配も同じ）
def grad_f(x):
    return A.T @ (A @ x - b)
```

### 5. 勾配法と劣勾配法の実装

滑らかな凸関数では、勾配法と劣勾配法は同じ更新式になります。  
ここでは「名前を変えて同じ式を2回書く」ことで、比較できるようにします。

```python
max_iter = 200
x0 = np.zeros(n)  # 初期点

# ステップサイズ（固定）
L = np.linalg.norm(A.T @ A, 2)  # リプシッツ定数の上界
t = 0.5 / L  # 安定するステップサイズ

# 履歴を保存するリスト
f_history_gd = []   # 勾配法
f_history_sub = []  # 劣勾配法

x_gd = x0.copy()
x_sub = x0.copy()

for k in range(max_iter):
    # 勾配法の更新
    x_gd = x_gd - t * grad_f(x_gd)
    f_history_gd.append(f(x_gd))
    
    # 劣勾配法の更新（滑らかなので g_k = grad_f(x_sub)）
    g_sub = grad_f(x_sub)  # 劣勾配集合は {grad_f(x_sub)} のみ
    x_sub = x_sub - t * g_sub
    f_history_sub.append(f(x_sub))
```

### 6. 収束の可視化

```python
fig, ax = plt.subplots(1, 1, figsize=(8, 5))

ax.semilogy(f_history_gd, label="Gradient Descent", linewidth=2)
ax.semilogy(f_history_sub, label="Subgradient Method", linestyle="--", linewidth=2)

ax.set_xlabel("Iteration")
ax.set_ylabel("Objective value $f(x_k)$")
ax.set_title("Convergence: Gradient Descent vs Subgradient Method")
ax.legend()
ax.grid(True, alpha=0.3)

plt.tight_layout()
plt.show()
```

滑らかな凸関数では勾配＝劣勾配なので、両者の曲線は完全に重なります（数値誤差レベルを除いて）。  
これが「滑らかな凸関数では勾配法も劣勾配法も同じ更新になる」ことの確認です。

__補足：非滑らかな関数での違い（参考）__

もし L1ノルムのような非滑らかな関数で比較したい場合は、

- 勾配法：そのままは使えない（微分不可能な点がある）
- 劣勾配法：劣勾配集合から一つ選んで更新

という違いが出ます。  
その場合は、L1ノルムの劣勾配（sign関数）を実装し、勾配法の代わりに近接勾配法などと比較することになります。


## まとめ

- 滑らかな凸関数では、勾配法と劣勾配法は同じ更新式になる。
- 上記コードでは、両アルゴリズムを同じ式で実装し、目的関数値の収束曲線が一致することを確認できる。
- 非滑らかな関数では、勾配法は使えず劣勾配法（あるいは近接勾配法）が必要になる。

このコードを実行すると、両者の収束曲線が重なる様子が視覚的に確認できます。


## 劣勾配の導出

L1 ノルム最小化問題

\[
f(X) = \|AX - b\|_1
\]

の劣勾配（劣微分）を導出します。  
ここではベクトル値関数として扱い、後で行列 \(X\) に戻します。

---

### 1. ベクトル関数としての書き換え

まず、\(AX - b\) をベクトル \(y \in \mathbb{R}^m\) とおきます：

\[
y = AX - b,\quad f(X) = \|y\|_1.
\]

\(y\) の各成分を \(y_i\) とすると、

\[
\|y\|_1 = \sum_{i=1}^m |y_i|.
\]

したがって、\(f(X)\) は

\[
f(X) = \sum_{i=1}^m |y_i|,\quad y_i = (AX - b)_i
\]

という合成関数として見ることができます。

---

### 2. 絶対値関数の劣微分

まず、1 変数関数 \(g(t) = |t|\) の劣微分を確認します。

- \(t > 0\) のとき：\(g(t) = t\) なので通常の微分で \(\partial g(t) = \{1\}\)。
- \(t < 0\) のとき：\(g(t) = -t\) なので \(\partial g(t) = \{-1\}\)。
- \(t = 0\) のとき：劣微分の定義より、任意の \(v\) について
  \[
  |s| \ge v s \quad (\forall s \in \mathbb{R})
  \]
  を満たす \(v\) の集合が \(\partial g(0)\) であり、これは \(v \in [-1, 1]\) です。

したがって、

\[
\partial |t| =
\begin{cases}
\{1\}, & t > 0,\\
[-1, 1], & t = 0,\\
\{-1\}, & t < 0.
\end{cases}
\]

符号関数 \(\mathrm{sign}(t)\) を

\[
\mathrm{sign}(t) =
\begin{cases}
1, & t > 0,\\
[-1, 1], & t = 0,\\
-1, & t < 0
\end{cases}
\]

と定義すると（0 での値は集合として扱う）、

\[
\partial |t| = \mathrm{sign}(t)
\]

と書けます。

---

### 3. ベクトル L1 ノルムの劣微分

多変数関数 \(h(y) = \|y\|_1 = \sum_{i=1}^m |y_i|\) の劣微分は、各成分の劣微分の直積になります：

\[
\partial h(y) = \partial |y_1| \times \partial |y_2| \times \dots \times \partial |y_m|.
\]

したがって、

\[
\partial h(y) = \mathrm{sign}(y_1) \times \mathrm{sign}(y_2) \times \dots \times \mathrm{sign}(y_m).
\]

これをベクトルとして

\[
\partial h(y) = \mathrm{sign}(y)
\]

と書きます（ここで \(\mathrm{sign}(y)\) は成分ごとの符号関数の集合）。

---

### 4. 合成関数の劣微分（連鎖則）

いま、\(y = AX - b\) はアフィン写像なので、合成関数 \(f(X) = h(AX - b)\) の劣微分は、劣微分の連鎖則より

\[
\partial f(X) = A^\top \partial h(AX - b)
\]

と書けます（行列 \(A\) の転置が現れるのは、線形写像の随伴作用素に対応します）。

ここに \(h(y) = \|y\|_1\) の劣微分 \(\partial h(y) = \mathrm{sign}(y)\) を代入すると、

\[
\partial f(X) = A^\top \mathrm{sign}(AX - b)
\]

が得られます。

---

### 5. 行列 \(X\) の場合の補足

もし \(X\) が行列で、\(AX\) が行列積を表す場合でも、考え方は同じです。  
\(AX - b\) をベクトル化して \(y = \mathrm{vec}(AX - b)\) とおき、\(A\) を対応する線形作用素とみなせば、同じ連鎖則により

\[
\partial f(X) = A^\top \mathrm{sign}(AX - b)
\]

が成り立ちます（ここで \(A^\top\) は線形写像としての随伴に対応し、結果は行列 \(X\) と同じサイズの劣勾配集合になります）。

---

以上より、目的関数 \(f(X) = \|AX - b\|_1\) の劣勾配は

\[
\partial f(X) = A^\top \mathrm{sign}(AX - b)
\]

と導出されます。
