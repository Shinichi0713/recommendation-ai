## 原因と考えられること考察

ご質問の状況は、**「RPCAのモデル化仮定と、UAV空撮画像の現実的な変動パターンが噛み合っていない」**ことが主な原因と考えられます。
以下、原因の整理と、RPCAの特性を踏まえた対策の方向性をまとめます。

### 1. RPCAの前提と空撮画像の現実

RPCA（Robust Principal Component Analysis）は、ざっくり言うと

- **低ランク成分**（背景・共通パターン）
- **スパース成分**（異常・ノイズ）

にデータ行列を分解する手法です。

典型的な仮定は：

- 背景は**時間的に安定**しており、**低ランク**で近似できる
- 異常は**空間的に局所**かつ**出現頻度が低い**（スパース）

というものです。

一方、UAV空撮画像では：

- カメラの**揺れ・回転・ズーム**（微小な動き）
- 太陽位置や雲による**照明変化**
- 風による**植生の揺れ**、水面の**きらめき**
- 建物・道路・植生の**テクスチャの不均一性**

など、**背景自体が時間的にかなり変動**します。これらはRPCAの「低ランクな背景」という仮定から外れやすく、結果として：

- 「背景の変動」が**異常側（スパース成分）に回りやすく**なる
- そのため**偽陽性（検出すべきでない変化）が増える**
- 逆に、**小さな異常は背景変動に埋もれて検出されない**

という問題が起きやすくなります。

### 2. 具体的な原因候補

__(1) 背景の変動が大きすぎて「低ランク」仮定が破綻__

- カメラの**微小な回転・平行移動**だけで、画素レベルでは大きな変化に見える
- 雲の影や太陽位置の変化で、**照度が大きく変わる**
- 植生の揺れや水面の反射で、**テクスチャが時間的にかなり変動**

これらは「異常」ではなく「背景の自然な変動」ですが、RPCAはそれらを**異常側（スパース成分）に押し出しやすい**です。その結果：

- 「可検出の多さ」＝**偽陽性が大量に発生**
- 一方で、**小さな異常は背景変動に埋もれて検出されない**

という状況になりがちです。

__(2) 異常のスケール・頻度が「スパース」仮定から外れている__

RPCAは「異常は**まれ**かつ**局所的**」という仮定で設計されていますが、例えば：

- 広範囲に広がる**影**や**水たまり**
- 連続フレームにわたる**工事エリア**や**車列**
- 広い範囲の**植生変化**（例：一斉に色が変わる）

などは、

- **空間的に広い**（＝スパースでない）
- **時間的に長く続く**（＝スパースでない）

ため、RPCAでは**異常として検出しにくい**か、あるいは**背景側に吸収されてしまう**ことがあります。

__(3) 画像前処理・正規化が不十分__

- コントラスト・明るさの正規化
- カメラジオメトリ（回転・平行移動）の補正
- 画素値のスケーリング（0–1 や標準化）

などが不十分だと、**背景の変動が強調**され、RPCAの分解が不安定になります。

## 3. RPCAの特性を踏まえた対策の方向性

__(1) 背景の「低ランク性」を保つための前処理__

- **画像レジストレーション（位置合わせ）**

  - 特徴点マッチングやオプティカルフローで、フレーム間の**アフィン変換**を推定し、画像を整列させる
  - これにより、カメラの揺れ・回転による変動を大幅に減らせる
- **照明正規化**

  - ヒストグラム均一化、ガンマ補正、ローカルコントラスト正規化などで、**照度変化の影響を抑える**
  - 影や雲の影響を背景に押し込めやすくなる
- **空間ピラミッド・マルチスケール処理**

  - 解像度を落とした低解像度画像でRPCAをかけることで、**細かいテクスチャ変動を平滑化**し、低ランク近似をしやすくする
  - その後、高解像度で局所的な異常を検出する、といった多段階処理も有効

__(2) RPCAのパラメータ・モデル拡張__

- **低ランク性の強さ（λ）** や **スパース性の強さ** を調整

  - 背景変動が大きい場合：低ランク側を**やや緩く**（ランクを少し高めに許容）
  - 偽陽性が多い場合：スパース性を**強く**して、小さな変化を無視する
- **重み付きRPCA / 空間・時間重み**

  - 特定の領域（例：常に揺れる木々、水面）には**異常スコアの重みを下げる**
  - 逆に、監視対象領域（道路、構造物）には**重みを上げる**
  - これにより、「検出すべきでない変化」を抑制し、「検出したい変化」を強調できる
- **オンラインRPCA / インクリメンタルRPCA**

  - 背景が時間とともにゆっくり変化する場合、**ウィンドウをスライド**させて背景を更新
  - 古いフレームを捨て、新しいフレームを取り込むことで、**背景モデルを適応的に更新**できる

__(3) RPCA以外の手法との組み合わせ__

- **オプティカルフロー＋差分**

  - まずオプティカルフローで**カメラ運動と背景運動を推定**
  - その上で、**予測される動きからの残差**をRPCAで解析する
  - これにより、背景の動きを事前に除去できる
- **教師あり／半教師あり異常検知との併用**

  - RPCAで**候補領域**を抽出
  - その後、CNNやオートエンコーダなどで**異常かどうかを再判定**
  - これにより、RPCAの「偽陽性」を後段でフィルタリングできる
- **時系列モデリング**

  - 各画素・パッチの時系列をLSTMや時系列AEでモデリングし、**予測からの外れ**を異常とみなす
  - RPCAは空間的な低ランク性を利用するが、時系列モデルは**時間的なパターン**を利用するため、補完的な情報が得られる

### 4. まとめ

- **「検知できない異常」と「可検出の多さ」** は、RPCAの

  - 低ランク背景仮定
  - スパース異常仮定
    が、UAV空撮画像の**現実的な変動パターン**と噛み合っていないことが主因と考えられます。
- 対策としては、

  1. **前処理（レジストレーション・照明正規化）**で背景を「低ランクに近づける」
  2. **RPCAパラメータ・重み付きモデル**で偽陽性を抑制
  3. **オンライン更新・マルチスケール処理**で背景の時間変化に対応
  4. **オプティカルフローや時系列モデルなど、他手法との組み合わせ**で補完

といった方向が有効です。

もし具体的なデータ（画像サイズ、フレーム数、撮影条件、異常の種類など）を教えていただければ、より踏み込んだパラメータ設計や前処理の提案も可能です。

追加情報ありがとうございます。「異常はほんの一部で間違いなくスパース」という状況であれば、**RPCAの「スパース異常」仮定は満たしている**と考えられます。その場合、問題はむしろ

- **低ランク側（背景）の分離がうまくいっていない**
- **マルチスペクトル（6波長）情報の扱い方がRPCAの仮定と噛み合っていない**

あたりにありそうです。

以下、可能性の高い原因と対策の方向性を整理します。

---

## 1. 低ランク背景の分離がうまくいっていない可能性

### (1) ランクの取り方が不適切

RPCAでは、背景を低ランクで近似しますが、

- ランクを**小さく取りすぎる**と、背景の変動まで「背景」に押し込められ、**異常が背景に埋もれる**
- ランクを**大きく取りすぎる**と、本来は異常側に出るべき変動まで「背景」として扱われ、**偽陽性が増える**

というトレードオフがあります。

「SVDをしても十分な特徴の分離がされているかはわからない」とのことなので、

- 特異値の減衰を見て、**どこまでを「背景」とみなすか**を再検討
- 実データで**背景のみのシーン**を用意し、そのSVDの特異値分布を確認

といった分析が有効です。

### (2) マルチスペクトル情報の扱い

6波長分のデータをどのように行列化しているかにも依存しますが、例えば

- 各波長を**別チャネルとして縦に並べる**（＝チャネル方向を追加次元として扱う）
- 各波長ごとに**別々の行列としてRPCAをかける**

など、設計によって低ランク性の出方が変わります。

もし**波長間の相関が強い**（＝同じ物体が似たスペクトルを持つ）場合、

- 波長方向も含めて**1つの低ランク行列**として扱うと、背景のランクが不必要に大きくなり、異常が埋もれやすくなる
- 逆に、波長ごとに**独立にRPCA**をかけると、背景の低ランク性が保たれやすくなる

といった違いが出ます。

---

## 2. マルチスペクトル特有の原因候補

### (1) 波長ごとのノイズ・キャリブレーション差

- 各波長で**センサ感度・ノイズ特性・レンズ透過率**が異なる
- キャリブレーション（暗電流補正、フラットフィールド補正）が不十分だと、**波長ごとに系統的なバイアス**が残る

これらは「背景の変動」としてRPCAに現れ、異常検出を妨げます。

**対策例：**

- 各波長ごとに**オフセット・ゲイン補正**を行う
- 基準板（白板・グレースケール）を用いた**放射較正**を行い、波長間のスケールを揃える
- 波長ごとに**平均・分散を正規化**してからRPCAに入力する

### (2) スペクトル特性の時間変化

- 太陽高度や大気条件（エアロゾル、水蒸気）の変化で、**各波長の反射・散乱特性が時間とともに変わる**
- これは「背景の時間変化」として低ランク側に現れ、異常検出を難しくします。

**対策例：**

- 撮影条件（太陽高度、気象）を揃えた**同一条件シーケンス**のみを解析対象にする
- 時系列で**背景のスペクトル変化をモデル化**し、その残差に対してRPCAをかける（例：時系列回帰＋RPCA）

---

## 3. データ前処理・正規化の問題

### (1) 画素値のスケーリング

RPCAは、基本的に**ユークリッド距離（L2ノルム）**に基づく分解です。そのため、

- 画素値の**ダイナミックレンジが波長ごとに大きく異なる**
- ある波長の値が他より**桁違いに大きい**

といった状況だと、その波長の変動が支配的になり、他の波長の異常が埋もれます。

**対策例：**

- 各波長ごとに**0–1スケーリング**または**標準化（平均0、分散1）**を行う
- あるいは、**ロバストなスケーリング**（例：中央値とMADで標準化）を用いる

### (2) 空間・時間の正規化

- 空間的に**明るさが不均一**（レンズビネット、太陽の方向による影）な場合、その変動が背景側に回り、異常を隠す
- 時間的に**照度が大きく変動**する場合も同様

**対策例：**

- 各フレームごとに**ローカルコントラスト正規化**を行う
- 背景のみのフレームから**平均背景画像**を作り、各フレームから差し引く（単純な背景差分）→ その残差にRPCAをかける

---

## 4. RPCAのパラメータ・モデル設計の問題

### (1) λ（低ランク vs スパースのトレードオフ）の設定

RPCAでは、低ランク項とスパース項のバランスを決めるハイパーパラメータ λ があります。

- λ が**小さすぎる** → スパース項が強くなり、**小さな変動も異常扱い** → 偽陽性増加
- λ が**大きすぎる** → 低ランク項が強くなり、**異常まで背景に押し込められる** → 検出漏れ増加

「検知できない異常」と「可検出の多さ」が同時に起きている場合、

- まずは**λを少し小さく**して、異常がスパース側に出るか確認
- その上で、偽陽性が増えすぎるようなら、**前処理や後処理でフィルタリング**する

というアプローチが現実的です。

### (2) 重み付きRPCAの利用

- 特定の波長・特定の空間領域に**重みを付ける**ことで、
  - ノイズの多い波長・領域は「異常」として扱いにくくする
  - 逆に、監視対象の波長・領域は「異常」として扱いやすくする

といった調整が可能です。

---

## 5. 確認すべきポイント（簡易チェックリスト）

1. **SVDの特異値分布**

   - 背景のみのデータでSVDを実行し、特異値の減衰を確認
   - どこまでを「背景」とみなすか（ランクの選択）を再検討
2. **波長ごとの正規化**

   - 各波長で平均・分散が大きく異なっていないか確認
   - 必要なら標準化や0–1スケーリングを適用
3. **背景のみのシーケンスでのRPCA**

   - 異常がないと分かっているシーケンスでRPCAを実行
   - スパース成分がほぼゼロになるか確認（＝背景がきちんと低ランクに分離されているか）
4. **λのスイープ**

   - λをいくつか変えて、異常検出性能と偽陽性のトレードオフを確認
   - 実データに合わせて最適なλを探索

---

- 「異常は間違いなくスパース」であれば、**RPCAのスパース異常仮定は満たしている**と考えられます。

- **λの値**や**ランクの決め方**
- 代表的な**異常の種類**（大きさ・スペクトル特性）

などを教えていただければ、より具体的なパラメータ設計や前処理の提案ができます。

以下、原因と対策を整理します。

---

## 1. 問題の整理

### 1-1. RPCAの前提
- **低ランク（L）**：背景は「ゆっくり変化する」「相関が高い」成分。
- **スパース（S）**：異常は「ごく一部の画素だけ」「独立に発生」する成分。

### 1-2. 実際の画像特性
- **木の葉の反射光が強い**  
  → 背景（L）が「強い局所コントラスト」を持ち、低ランク近似が難しい。
- **葉の色が異なる部分はスパースだが、領域としてはそこそこ大きい**  
  → 異常（S）は「画素数的にはスパース」だが、「連続した領域」として存在するため、**空間的にまとまったスパース**になっている。

このズレが、
- **検知できない異常（偽陰性）**：異常が背景に埋もれる
- **可検出の多さ（偽陽性）**：背景の変動がスパース側に漏れる
という問題を引き起こしています。

---

## 2. 原因の具体化

### 2-1. 反射光の強さ・局所コントラスト
- 強い反射光は、**局所的に大きな輝度差**を作ります。
- RPCAの低ランク近似は「大局的な相関」を捉えるため、**局所的な強いコントラストはうまく表現できず、スパース側に漏れやすい**です。

### 2-2. 異常の「スパース性」と「領域サイズ」
- RPCAのスパース性は「画素単位のスパース」を想定しています。
- しかし、実際の異常は「小さなパッチ」や「連続した領域」として現れるため、**空間的にまとまったスパース**になります。
- これにより、
  - 異常が背景に埋もれやすい（偽陰性）
  - 背景の変動がスパースとして検出されやすい（偽陽性）
  という問題が起きます。

### 2-3. マルチスペクトル（6波長）の扱い
- 6波長をそのまま行列化すると、**チャネル方向の相関**が強くなり、低ランク側に異常が吸収されやすくなります。
- 波長ごとに正規化・キャリブレーションが不十分だと、**反射光の影響がチャネル間で不均一**になり、分離が難しくなります。

---

## 3. 具体的な対策

### 3-1. 前処理の強化

#### (1) 局所コントラスト正規化
- 強い反射光の影響を抑えるため、**局所的な輝度正規化**を行います。
  - 例：ガンマ補正、ヒストグラム均一化、CLAHE（Contrast Limited Adaptive Histogram Equalization）など。
- 目的：背景を「より低ランクに近い」状態にすること。

#### (2) マルチスペクトルの正規化・キャリブレーション
- 各波長ごとに、
  - オフセット補正（暗電流など）
  - ゲイン補正（センサー感度）
  - 反射率への変換（可能なら）
  を行い、**チャネル間のバランスを揃える**ことが重要です。

#### (3) 画像レジストレーションの精度向上
- UAVの揺れや視点変化による位置ずれは、**低ランク分離を大きく妨げます**。
- 特徴点マッチングやオプティカルフローで、**サブピクセル精度のレジストレーション**を行い、背景の相関を高めます。

---

### 3-2. RPCAのパラメータ・手法の調整

#### (1) λ（スパース性の重み）の調整
- λが大きすぎると、**異常が背景側に吸収されやすく**なります（偽陰性）。
- λが小さすぎると、**背景の変動がスパース側に漏れやすく**なります（偽陽性）。
- 実際のデータで、**Sのヒストグラムや空間分布**を見ながら調整するのが有効です。

#### (2) 重み付きRPCA（Weighted RPCA）
- 空間的に「異常が起こりやすい領域」に重みを付けることで、
  - 異常の検出感度を上げる
  - 背景の変動を抑制する
  ことができます。
- 例：葉の色が異なる領域をマスクとして与え、その領域のスパース重みを強める。

#### (3) マルチスケールRPCA
- 画像を複数のスケール（解像度）に分解し、それぞれでRPCAを適用します。
- 粗いスケールで大局的な背景を捉え、細かいスケールで局所的な異常を捉えることで、**空間スケールの違いを分離**できます。

---

### 3-3. 他手法との組み合わせ

#### (1) オプティカルフロー＋差分
- RPCAの前に、**オプティカルフローで動きを補償**し、差分画像を作ります。
- 動きが補償された差分は、**背景の変動を抑えつつ異常を強調**できるため、RPCAの入力として有効です。

#### (2) 時系列モデリング（ARモデルなど）
- 各画素の時系列をモデル化し、**予測残差**を異常度として使います。
- RPCAと組み合わせることで、**時間的な異常**と**空間的な異常**を分離できます。

#### (3) 教師あり・半教師あり学習
- もし「正常な葉」と「異常な葉」のサンプルがあれば、
  - 特徴量（色、テクスチャ、スペクトル）を抽出
  - 分類器（SVM、Random Forest、CNNなど）で異常検出
  を行うことも有効です。
- RPCAの結果を特徴量の一つとして組み込むこともできます。

## 4. まとめ

- **木の葉の反射光が強い**場合、RPCAの低ランク背景仮定と現実の画像特性がズレやすく、  
  - 異常が背景に埋もれる（偽陰性）
  - 背景の変動がスパースとして検出される（偽陽性）
  という問題が起きやすいです。

- 対策としては、
  1. **前処理の強化**（局所コントラスト正規化、マルチスペクトル正規化、高精度レジストレーション）
  2. **RPCAのパラメータ・手法の調整**（λの調整、重み付きRPCA、マルチスケールRPCA）
  3. **他手法との組み合わせ**（オプティカルフロー＋差分、時系列モデリング、教師あり学習）
  が有効です。

もし、具体的な画像サイズ、フレーム数、異常の種類（病気、虫害、枯れなど）や、使用しているRPCAの実装（λの値、ランク選択方法など）を教えていただければ、より踏み込んだ提案が可能です。

はい、**木が生えていない領域での異常検出精度が悪い場合、領域をパッチング（小領域に分割）することは非常に有効**です。

理由と具体的な方法を整理します。

---

## 1. なぜパッチングが有効か

### 1-1. RPCAの前提と「木がない領域」の特性

- RPCAは「**背景は低ランク**」という前提で動作します。
- 木が生えている領域：
  - 葉や枝が複雑に重なり、**テクスチャが豊富**で、背景が「ある程度のランク」を持ちやすい。
  - 結果として、**低ランク背景とスパース異常の分離が比較的うまくいく**ことが多い。
- 木が生えていない領域（例：土、草地、裸地）：
  - テクスチャが単調で、**背景がほぼ1ランク（平坦）に近い**。
  - わずかなノイズや照明変動が、**スパース側に漏れやすく**、偽陽性が増えやすい。

### 1-2. パッチングの効果

- 画像全体を一つの行列として扱うと、**木がある領域の強い変動が全体の低ランク推定に影響**し、木がない領域の分離が悪化します。
- パッチング（小領域に分割）すると、
  - **木があるパッチ**：背景が複雑で、RPCAが比較的安定。
  - **木がないパッチ**：背景が単純で、RPCAが不安定だが、**パッチ単位でパラメータを調整できる**。
- これにより、**領域ごとに最適なλ（スパース重み）やランクを設定**できるため、木がない領域での精度改善が期待できます。

---

## 2. パッチングの具体的な方法

### 2-1. 単純なグリッド分割

- 画像を一定サイズのパッチ（例：64×64、128×128）に分割します。
- 各パッチに対して独立にRPCAを適用します。
- メリット：
  - 実装が簡単。
  - パッチごとにλやランクを変えやすい。
- デメリット：
  - パッチ境界で不連続が生じる可能性がある。

### 2-2. オーバーラップ付きパッチング

- パッチを重ねて分割します（例：50%オーバーラップ）。
- 各画素が複数のパッチに属するため、**境界の不連続を緩和**できます。
- 最終的な異常マップは、パッチ結果を平均または最大値で合成します。

### 2-3. セマンティックセグメンテーションに基づくパッチング

- 事前に「木がある領域」「木がない領域」をセグメンテーションし、
  - 木がある領域：大きなパッチ or 全体として処理
  - 木がない領域：小さなパッチ or 別のパラメータで処理
  といった**領域ごとの戦略**を取ることも有効です。

---

## 3. パッチング時のパラメータ調整

### 3-1. 木がないパッチでのλ調整

- 木がないパッチは背景が単純なため、**λを大きくする（スパース性を強くする）**ことで、ノイズのスパース漏れを抑制できます。
- 例：
  - 木があるパッチ：λ = 1.0
  - 木がないパッチ：λ = 1.5 ～ 2.0

### 3-2. ランクの調整

- 木がないパッチでは、背景ランクを**1または2に固定**するなど、低めに設定すると安定します。
- 木があるパッチでは、データに応じてランクを自動選択（特異値の減衰を見る）するのが良いでしょう。

### 3-3. 正規化の強化

- 木がないパッチでは、**局所コントラスト正規化**を強めにかけることで、照明変動の影響を抑えられます。
- 例：パッチごとにヒストグラム均一化やCLAHEを適用。

---

## 4. パッチング以外の補完策

### 4-1. マスク付きRPCA

- 「木がない領域」をマスクとして与え、**その領域だけ別のλや正規化を適用**する方法もあります。
- パッチングと組み合わせることで、より柔軟な制御が可能です。

### 4-2. 木がない領域専用の前処理

- 木がない領域に対してのみ、
  - 時系列の平滑化（移動平均など）
  - 空間的な平滑化（ガウシアンフィルタ）
  を適用し、**背景をより低ランクに近づける**ことも有効です。

局所コントラスト正規化（Local Contrast Normalization）は、**画像の各画素の値を、その周辺（局所領域）の統計量（平均・分散など）を使って正規化する処理**です。

目的は、
- **照明変動や影の影響を抑える**
- **テクスチャやエッジを強調する**
- **画像全体のコントラストを均一にする**
ことです。

---

## 1. 基本的な考え方

画像の各画素 \(I(x,y)\) に対して、その周辺（例：7×7や15×15のウィンドウ）の画素値から、

- 局所平均 \(\mu(x,y)\)
- 局所標準偏差 \(\sigma(x,y)\)

を計算し、以下のように正規化します：

\[
I_{\text{norm}}(x,y) = \frac{I(x,y) - \mu(x,y)}{\max(\sigma(x,y), \epsilon)}
\]

- \(\epsilon\) はゼロ除算を防ぐための小さな値（例：\(10^{-6}\)）。
- これにより、各画素は**局所的に平均0、分散1**に近づきます。

---

## 2. なぜRPCAに有効か

### 2-1. 照明変動・影の影響を抑える

- UAV空撮では、太陽の位置や雲の影響で**照明が時間的に変動**します。
- 局所コントラスト正規化を行うと、
  - 明るい領域も暗い領域も、**局所的には同じ統計特性**になります。
  - 結果として、**時間的な照明変動が背景（低ランク側）に吸収されやすく**なります。

### 2-2. テクスチャ・エッジを強調する

- 木の葉や枝のテクスチャ、異常な色の変化などは、**局所的なコントラスト**として現れます。
- 正規化により、これらの特徴が**相対的に強調**され、RPCAのスパース成分（異常）として検出されやすくなります。

### 2-3. 背景を「より低ランクに近づける」

- 照明変動が抑えられると、背景の時間変化が**より滑らか**になります。
- これにより、RPCAの低ランク背景仮定と実際の画像が噛み合いやすくなり、**分離性能が向上**します。

---

## 3. 具体的な実装例（Python＋OpenCV）

### 3-1. 単純な局所コントラスト正規化

```python
import cv2
import numpy as np

def local_contrast_normalization(image, window_size=15, epsilon=1e-6):
    """
    image: 入力画像（グレースケール or 各チャネルごとに処理）
    window_size: 局所ウィンドウのサイズ（奇数推奨）
    epsilon: ゼロ除算防止用の小さな値
    """
    # ガウシアンカーネルで局所平均を計算
    mu = cv2.GaussianBlur(image.astype(np.float32), (window_size, window_size), 0)
    
    # 局所分散を計算
    sigma_sq = cv2.GaussianBlur((image.astype(np.float32) - mu)**2, (window_size, window_size), 0)
    sigma = np.sqrt(sigma_sq)
    
    # 正規化
    normalized = (image.astype(np.float32) - mu) / np.maximum(sigma, epsilon)
    
    # 値を0～255にクリップ（必要に応じて）
    normalized = np.clip(normalized, -3, 3)  # 外れ値抑制
    normalized = (normalized - normalized.min()) / (normalized.max() - normalized.min()) * 255
    return normalized.astype(np.uint8)
```

### 3-2. マルチスペクトル画像への適用

6波長のマルチスペクトル画像の場合、**各チャネルごとに独立に正規化**するのが一般的です。

```python
# image_shape: (H, W, C) のNumPy配列
normalized_channels = []
for c in range(image.shape[2]):
    normalized_ch = local_contrast_normalization(image[:,:,c], window_size=15)
    normalized_channels.append(normalized_ch)

normalized_image = np.stack(normalized_channels, axis=2)
```

```python
clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8,8))
```

```python
    mu = cv2.GaussianBlur(image.astype(np.float32), (window_size, window_size), 0)
    
    # 局所分散を計算
    sigma_sq = cv2.GaussianBlur((image.astype(np.float32) - mu)**2, (window_size, window_size), 0)
    sigma = np.sqrt(sigma_sq)
    
    # 正規化
    normalized = (image.astype(np.float32) - mu) / np.maximum(sigma, epsilon)
    
    # 値を0～255にクリップ（必要に応じて）
    normalized = np.clip(normalized, -3, 3)  # 外れ値抑制
    normalized = (normalized - normalized.min()) / (normalized.max() - normalized.min()) * 255
    return normalized.astype(np.uint8)
```

### 3-2. マルチスペクトル画像への適用

6波長のマルチスペクトル画像の場合、**各チャネルごとに独立に正規化**するのが一般的です。

```python
# image_shape: (H, W, C) のNumPy配列
normalized_channels = []
for c in range(image.shape[2]):
    normalized_ch = local_contrast_normalization(image[:,:,c], window_size=15)
    normalized_channels.append(normalized_ch)

normalized_image = np.stack(normalized_channels, axis=2)
```

## 4. バリエーション

### 4-1. CLAHE（Contrast Limited Adaptive Histogram Equalization）

- OpenCVの `cv2.createCLAHE()` は、**局所ヒストグラム均一化**を行う強力な手法です。
- コントラストの上限を設けることで、ノイズ増幅を抑えつつ局所コントラストを改善します。

```python
clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8,8))
normalized = clahe.apply(image)
```

### 4-2. 白黒化（Whitening）との関係

- 局所コントラスト正規化は、**局所的な白黒化（Whitening）**に近い処理です。
- 機械学習では、入力データを白黒化することでモデルの学習が安定することが知られており、RPCAにも同様の効果が期待できます。

## 5. RPCA前処理としての位置づけ

- **入力画像 → 局所コントラスト正規化 → RPCA**
- 正規化により、
  - 背景がより低ランクに近づく
  - 異常がスパース成分として現れやすくなる
- 特に、**木の葉の反射光が強い場合**や**照明変動が大きい場合**に有効です。

## 6. まとめ

- 局所コントラスト正規化は、**各画素をその周辺の統計量で正規化する処理**です。
- 目的は、
  - 照明変動・影の影響を抑える
  - テクスチャや異常を強調する
  - 背景をより低ランクに近づける
  ことです。
- RPCAの前処理として適用すると、**異常検出精度の向上**が期待できます。
