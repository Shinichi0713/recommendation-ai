
## 線形代数の役割

線形代数を勉強していても、そもそも線形代数の処理にどんな意味があるか理解していないと、唯の処理となってしまいます。
線形代数を"唯の処理"にしないために、知っておくと理解が進む線形代数の概念について説明します。


### 1. 行列を「数字の表」ではなく「動き」と捉える

多くの人が挫折するのは、行列をただの「数字が並んだ箱」だと思って計算練習ばかりするからです。

* **本質:** 行列はベクトルを別の場所に飛ばす **「写像（関数）」** です。
* **イメージ:** * 2倍に引き伸ばす動き
* 90度回転させる動き
* ペシャンコに押しつぶす動き
* **メリット:** これを知ると、「行列の掛け算」が「動きの合成」に見えてきます。

>__写像とは__  
>線形代数における「写像（Mapping）」を一言で言うなら、 **「あるベクトルを、別のベクトルへと変換する『ルール』」** のことです。
>プログラミングをされる方なら、 **「ベクトルを入力として受け取り、新しいベクトルを返す関数」** と考えると非常に理解しやすくなります。  
>__1. イメージで捉える「写像」__  
>写像 $f$ がベクトル $\vec{x}$ を $\vec{y}$ に変えるとき、数学では $f(\vec{x}) = \vec{y}$ と書きます。線形代数の主役である「線形写像」は、この操作を**行列の掛け算**で行います。
>このとき、行列  は単なる数字の表ではなく、 **「空間全体をどう変形させるかという指示書」** になります。  
>__2. 線形写像の「2つの絶対ルール」__  
>ただの関数ではなく「線形」写像と呼ぶためには、空間を「まっすぐ」に保つ必要があります。
>* **加法性:** 足してから飛ばしても、飛ばしてから足しても同じ。
>* $f(\vec{u} + \vec{v}) = f(\vec{u}) + f(\vec{v})$
>* **斉次性（スカラー倍）:** 何倍かしてから飛ばしても、飛ばしてから何倍かしても同じ。
>* $f(c\vec{u}) = c f(\vec{u})$
>**直感的な意味:**
このルールを守る写像は、 **「グリッド（格子）が等間隔でまっすぐなまま、原点が固定された状態で変形する」** という性質を持ちます。グニャリと曲がったり、原点が移動したりすることはありません。
>__3. 具体的に「何をしているか」の例__  
>2次元の写像（2x2行列）で考えると、以下のような「動き」がすべて写像です。
>* **拡大・縮小:** 空間をビヨーンと伸ばす。
>* **回転:** 原点を中心にクルッと回す。
>* **鏡映:** 鏡に映したように反転させる。
>* **射影:** 3次元の影を2次元の床に落とす（情報を削る）。
>__4. なぜ「写像」という考え方が重要か？__  
>「行列 $A$ を計算する」と考えると苦行ですが、「写像 $f$ で空間を動かす」と考えると、以下のような線形代数の重要概念がスッと繋がります。
>* **逆行列:** 「逆の動き」をして元に戻せるか？（潰れた空間は戻せないので逆行列はない）
>* **核（Kernel）:** 写像によって「原点（ゼロ）」に押しつぶされてしまったベクトルたちはどこか？
>* **固有ベクトル:** 写像によって「向きが変わらず、長さだけが変わった」ラッキーな軸はどこか？


### 2. 「基底（座標系）」は履き替えることができる

私たちは無意識に $x, y$ 軸（標準基底）を使っていますが、線形代数の魔法は **「自分に都合の良い軸に勝手に変えていい」** という点にあります。

* **例え:** 地図で「北」を上にするのではなく、「進んでいる方向」を上にした方がナビは見やすいですよね。
* **対角化の意味:** 複雑な行列を扱うとき、軸を「固有ベクトル」の方向に履き替えると、行列の計算がただの「各軸の引き伸ばし」という極めて単純な作業に変わります。

### 3. 「次元」を情報の自由度と考える

「3次元」と言われたら図形を想像しますが、科学計算では **「変数の数」** を意味します。

* **ランク（階数）の重要性:** 100個の変数があっても、実は互いに関係し合っていて、本質的な情報が3つしかなければ、それは「3次元の現象」です。
* **イメージ:** 影（2次元）から元の物体（3次元）を推測するように、情報の「削ぎ落とされ方」を管理するのが線形代数の役割です。


## 学習の鍵

線形代数を理解する上で鍵となることをリストアップします。
本節で述べる点を留意の上で学習を進めていくと、理解が早いと思っています。

### A. 処理を可視化

数式だけで追うと必ず限界が来ます。
Pythonを使って行列による処理の前後を可視化してみてください。
可視化により処理の意味が理解出来ます。
結果、処理の意味について理解が深まります。

### B. 線形代数は数のブロックを写像している

線形代数とは、**数字のブロックを写像して別の塊にする処理**です。

例えると、「レシピ（写像）」と「材料（数字のブロック）」、あるいは　**「加工マシン（写像）」と「素材（数字のブロック）」**　という比喩を使って説明できるかもしれません。

__「写像」は、変化を起こす魔法のマシン__

ここで、行列 $A$ という **「写像」が登場します。写像とは、この数字のブロックを放り込むと、別の形に変えて出してくれる「加工マシン」** のようなものです。

$$A (\text{元のブロック}) = \text{新しいブロック}$$

__加工のイメージ__

「写像」をイメージしやすいように、3つの具体的な加工（処理）を説明します。

__① 「拡大・縮小」のマシン__

数字のブロックの中身を2倍にしたり、半分にしたりします。

実務: 画像のサイズ変更や、スピーカーから出る音量を大きくする処理。

__② 「回転」のマシン__

ブロックが表す空間の向きをクルッと変えます。

実務: スマホを横に傾けたとき、画面の表示を回転させる処理。

__③ 「情報の抽出」のマシン（射影）__

立体的なブロックを、ペシャンコに押しつぶして「影」にします。

実務: 膨大な顧客データから、「購買意欲」という重要な要素だけを抜き出すAIの処理。
